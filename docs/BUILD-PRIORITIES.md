# BUILD-PRIORITIES — слои сборки (лимит фич, ядро первым)

~88-90 фич к постройке (COMPLETE-FEATURE-MAP) — это много. НЕ строить все равномерно.
Лимитируем и приоритизируем по СЛОЯМ: сначала то, без чего продукт нельзя продать, потом полнота,
потом полировка. Цель: рабочий ПРОДАВАЕМЫЙ продукт быстрее, не утонуть в edge-фичах.

## Сначала разметить: каждой фиче в COMPLETE-FEATURE-MAP проставить тег P0/P1/P2
- **P0 — ЯДРО (MVP, без этого не продать).** То, что юзер трогает в первый день + путь монетизации.
- **P1 — ПОЛНОТА.** Остальные фичи столов, конфиг-окна, вторичные потоки.
- **P2 — ПОЛИРОВКА/ОТЛОЖЕНО.** Edge-фичи, аналитика, мастер-промпт-секции, редкие, parked.

## Порядок сборки = ПО СЛОЯМ (не всё подряд)
1. **СЛОЙ P0 по ядровым столам — ПЕРВЫМ (горизонтально, до продаваемого MVP):**
   - Разговор: чат-ядро (✅) · Усиление · модель/Авто · Веб-поиск · Длина (✅) · Глубоко
   - Студия: ХОТЯ БЫ 1 рабочий тип (картинки — самый ходовой) end-to-end
   - Память: recall/remember (✅) + RAG-grounding
   - Аккаунт/Биллinг: вход + баланс + пополнение (на экране уже «$4.20» → монетизация)
   - Настройки: базовые
   → На этом продукт уже РАБОТАЕТ и продаётся. Это веха №1.
2. **СЛОЙ P1 — полнота столов:** остальные типы Студии, Агенты/Agent-OS, Дизайн-канвас,
   Код/терминал/MCP, все конфиг-окна. Стол-за-столом до зелёного.
3. **СЛОЙ P2 — полировка:** аналитика, мастер-промпт-слой, диагностика, редкие фичи, parked.

## Правила эффективности (нудж — больше работы, меньше возни)
- **ЛОК счётчика:** трекер = COMPLETE-FEATURE-MAP (186/~88-90). **НЕ пересчитывать фичи заново** —
  считали уже 4 раза (95→139→196→186). Считать вредно, строить полезно.
- **P0 первым, P2 не трогать сейчас.** Не доводить edge-фичу до идеала, пока ядро не закрыто.
- **Каждый гир = полный цикл** (оценка→билд→тест→скрин→двойной критик→зелёное→коммит), но без
  лишних кругов: критик принял → СЛЕДУЮЩАЯ фича, не перепроверять без причины.
- **Бэкенд-врезка** — главная работа (UI/экраны есть, бэк не в проде). Фокус: domotai `/shell/*.js` → server.py.
- **Темп:** доклад бинарно после каждого гира, идти без остановок внутри слоя.

## Веха №1 (что делает продукт продаваемым)
Разговор P0 + Студия-картинки + Память + Биллинг-вход/баланс/пополнение = **первый продаваемый срез**.
Это цель прежде полноты. Потом P1 (полнота), потом P2 (полировка).
