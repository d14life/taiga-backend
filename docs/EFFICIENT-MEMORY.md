# Токен-экономное использование памяти + скиллов (ponytail-стиль)

Правило: спрашивай ТОЧЕЧНО, проси КОРОТКО, не дублируй. Память чтобы НЕ читать всё подряд,
а доставать ровно нужное. Самый дешёвый путь под каждую задачу:

## graphify — граф КОДА (что с чем связано)
- **`graphify path "A" "B"`** — кратчайшая связь между двумя узлами. Самое дешёвое, начинай с него.
- **`graphify explain "X"`** — узел + соседи, сфокусированно.
- **`graphify query "вопрос" --budget 600`** — ВСЕГДА с `--budget` (дефолт 2000 = много). Один точный
  вопрос лучше широкого. Для >5000 узлов — `--no-viz`.
- НЕ дампить весь граф, НЕ перезапрашивать то, что уже видел.

## gbrain — ПАМЯТЬ по докам (через MCP в аппе)
- **`gbrain search "..."`** — дешёвый ретрив: ранжированные сниппеты, БЕЗ LLM. Агент (Claude) сам
  синтезирует из них. Это путь по умолчанию.
- **Избегай `gbrain think`** — он тратит ЛИШНИЙ NVIDIA-LLM-вызов на prose-синтез. Нужен только если
  хочешь готовый цитируемый ответ отдельно от Claude. В аппе синтезируй сам.
- Через MCP агент получает сниппеты и рассуждает — не гоняй think зря.

## Внешний совет (tools/external-critique.py) — дорого, только на важное
- **LITE по умолчанию**: 2 быстрые модели (llama 70b+8b), кап 450 токенов. Дёшево.
- **`--synth-only`** — печатать ТОЛЬКО синтез (не три полотна советников). Меньше вывода.
- **`--deep`** — добавляет deepseek-r1 (reasoning, жжёт токены думая) ТОЛЬКО для критичных/сложных решений.
- Зови на ВАЖНЫХ шестерёнках/спорных решениях, НЕ на каждой мелочи.
  ```
  python3 tools/external-critique.py --synth-only "<вопрос>"        # дёшево
  python3 tools/external-critique.py --deep --file x.tsx "идеально?" # дорого, для критичного
  ```

## Скиллы и общая экономия
- **pick-skills** — раз на задачу, не дёргать повторно.
- **ponytail full** — режет лишний код в выводе (−22% токенов), держи включённым всю сборку.
- **opusplan** — Opus планирует, Sonnet делает дешёвую механику. Дорогую модель не на лёгкое.
- **Параллельные агенты / Workflow** — у них ИЗОЛИРОВАННЫЙ контекст; тяжёлое чтение туда, не в главный.
- **Не перечитывай** уже прочитанные файлы; не дампь файл целиком — точечный Read/grep.
- **Кап вывода** у любой модели-помощника (max_tokens) + «отвечай коротко, буллеты».

## Порядок дешевизны (бери самый дешёвый, что отвечает на вопрос)
grep/точечный Read  <  graphify path/explain  <  graphify query --budget  <  gbrain search  <
внешний совет LITE  <  gbrain think / совет --deep. Поднимайся по лестнице только если нужнее.
