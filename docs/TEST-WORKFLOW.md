# A-to-Z воркфлоу сборки + теста (слой за слоем, авто-петля)

Полный детерминированный поток. НЕ строить всё разом — строго слой за слоем,
каждый слой проходит авто-петлю до зелёного, и только потом следующий.

## Принцип: strangler-fig (вынимаем деталь, чистим, ставим обратно)
Не переписывать с нуля и не big-bang. Берём ОДНУ деталь (фичу/панель/эндпоинт),
выносим чисто, проверяем поведение байт-в-байт по FEATURE-INVENTORY, ставим обратно.
**Если деталь уже сделана и зелёная — не трогаем, идём дальше.** Так весь продукт
постепенно заменяется рабочими кусками, ничего не ломая.

## Гранулярность: ФУНДАМЕНТ → СТЕНЫ → ФИЧА ЗА ФИЧЕЙ (каждый бит до идеала)
Строим как дом: сначала фундамент (слой 1 каркас), потом стены (следующие слои), и ВНУТРИ
каждого слоя — ФИЧА ЗА ФИЧЕЙ, не всё разом. Одна фича доводится до ИДЕАЛА (зелёное + критик-агент
принял) → только потом следующая фича. Все фичи слоя идеальны → слой принят → следующий слой.

## Порядок слоёв = 5 СТОЛОВ + Ultra (решение Damir, DECISIONS-LOG 2026-06-18)
Топ-навигация = **5 столов + Ultra**. **МОДЫ ≠ УСИЛИЕ (Damir поправил):** Моды (Мозг/Совет/Дебаты/
Сравнить) = ОТДЕЛЬНЫЕ режимы (несколько моделей думают). Усилие = ОТДЕЛЬНЫЙ параметр (deep/native/
fast, нативно через API или подбором моделей). Контрол: модель+мод+усилие в одном UI.
**Источник истины каждого стола = эталон-репо (REFERENCE-REPOS.md) + ОБА дизайна (v3 + usertest).** Всегда.

0. ФУНДАМЕНТ: Shell + macOS-док (✅ Фича 1 готова, d7e3617 — пины переориентировать со старых мод-пиллов на 5 столов)
1. **Стол РАЗГОВОР (Чат)** — ядро чата + «Усиление» (Мозг/Совет/Дебаты/Сравнить/Relay как уровни)
2. **Стол СТУДИЯ** — медиа (картинки/видео/музыка/озвучка/3D), база = github.com/anil-matcha/open-generative-ai (+наше, ~7-10 режимов)
3. **Стол ДИЗАЙН** — дизайн-канвас
4. **Стол АГЕНТЫ** — Agent-OS / автономия / параллелизм
5. **Стол КОД** — терминал / MCP / скиллы / песочница
6. **Ultra** — всё-в-одном стол (универсал + эксклюзив: пересборка системы разговором)
СКВОЗНОЕ (по ходу столов): Память/RAG · Настройки/кастомизация · Polish.
ОТЛОЖЕНО (не сейчас): Платёж #1 · Фаза D масштаб (гейт перед публичным выкатом).

Один стол полностью зелёный → следующий. НИКОГДА не начинать стол N+1 пока N не принят.
Внутри стола: список фич (из FEATURE-LEDGER) → каждую до идеала по микро-петле ниже. База стола = эталон-репо.

## ТУЛЗЫ СБОРКИ (использовать в петле)
- **Память (вместо грепа/догадок):** `graphify path/explain/query --budget 600` (граф кода) · `gbrain search "..."` (доки). См. EFFICIENT-MEMORY.md.
- **Внешний критик-совет (на каждый бит):** `python3 tools/external-critique.py --synth-only "<вопрос>" --file <путь>` (LITE) · `--deep` (важное).
- **Свежесть памяти (после правок):** `bash tools/reindex-memory.sh` (graphify+gbrain+чистка _archive).
- **Регистратор:** решения/просьбы/находки → `docs/DECISIONS-LOG.md` (новое наверх) + `gbrain capture "..."`.
- **Планка:** docs/ENGINEERING-PRINCIPLES.md (11 frontier-стандартов). **Экономия:** ponytail full · opusplan.

## МИКРО-ПЕТЛЯ КАЖДОЙ ФИЧИ (loop-until-perfect — каждый бит до идеала)
Для КАЖДОЙ фичи слоя гнать этот цикл, пока критик-агент не примет (2 чистых прогона подряд).
Одна фича идеальна → следующая. Не строить следующую, пока текущая не безупречна:

```
ШАГ 0 — ОЦЕНКА существующего кода области (ДО стройки)
  • что уже есть для этого слоя? работает? чисто? нет stub/мёртвого/раздутого?
  • chat.tsx=8157 строк, server.py=15125 — кандидаты на декомпозицию
  • вердикт по детали: ОК-как-есть / почистить / декомпозировать / перебрать (strangler-fig)
  • сохранить рабочее поведение байт-в-байт; не слепо сохранять и не слепо переписывать

ШАГ A — СТРОЙ / ЧИСТКА (агент/воркеры)
  • fan-out: агент на каждую фичу слоя (dispatching-parallel-agents / Workflow)
  • ponytail full: минимальный код, без урезания валидации/безопасности
  • Sonnet на простую механику, Opus на сложную логику (opusplan)

ШАГ B — ВЕРИФИКАЦИЯ (на каждую фичу)
  • из taiga-web: npx tsc --noEmit  → 0 ошибок
  • трогал бэк: python3 -c "import ast;ast.parse(open('server.py').read())"
  • /app возвращает 200
  • смоук фичи: реально дёрнуть (клик/запрос), не «выглядит ок»
  • СКРИНШОТ фичи (Playwright / preview)

ШАГ C — ДВОЙНОЙ КРИТИК (внутренний агент + внешний совет) по ПЛАНКЕ ENGINEERING-PRINCIPLES
  • C1 ВНУТРЕННИЙ критик-агент (НЕ тот, что строил) — «почему НЕ идеально»:
    - визуал: скрин vs design-v3 + docs/screenshots/ + usertest-jun15
    - поведение: vs FEATURE-LEDGER — байт-в-байт, edge-кейсы, ошибки/пустые/лоадинг
    - качество по docs/ENGINEERING-PRINCIPLES.md (11 пунктов): простота/типы/все-состояния/
      безопасность/перф/a11y/читаемость/прод-готовность; мёртвое/заглушки/дубли/раздутость
    - на важном — 2-3 линзы параллельно (correctness/UX/security)
  • C2 ВНЕШНИЙ СОВЕТ (другая модель + веб, анти-само-суд): на КАЖДЫЙ бит LITE
    `python3 tools/external-critique.py --synth-only "идеальна ли фича X?" --file <путь>`
    на сложном/важном — `--deep`.
  • вердикт обоих: ИДЕАЛЬНО / список дефектов. Пока есть дефекты — фича НЕ принята.

ШАГ D — ФИКС (если критик нашёл ЧТО-УГОДНО)
  • fan-out фикс-агентов: агент на дефект
  • починили → ВЕРНУТЬСЯ к ШАГ B (та же фича)
  • два чистых прогона B+C подряд по ФИЧЕ = фича ИДЕАЛЬНА → следующая фича
  • все фичи слоя идеальны → слой ПРИНЯТ → следующий слой

ШАГ E — КОММИТ (только на зелёном)
  • сначала taiga-web, потом корень-указатель → push
  • один вынос = один коммит, никогда не коммитить сломанное
```

## A-to-Z ТЕСТ (после всех слоёв — полный прогон)

4. Финальный агент-критик «что упущено / не проверено / не идеально» → новый круг фиксов.
