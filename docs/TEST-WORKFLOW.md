# A-to-Z воркфлоу сборки + теста (слой за слоем, авто-петля)

Полный детерминированный поток. НЕ строить всё разом — строго слой за слоем,
каждый слой проходит авто-петлю до зелёного, и только потом следующий.

## Принцип: strangler-fig (вынимаем деталь, чистим, ставим обратно)
Не переписывать с нуля и не big-bang. Берём ОДНУ деталь (фичу/панель/эндпоинт),
выносим чисто, проверяем поведение байт-в-байт по FEATURE-INVENTORY, ставим обратно.
**Если деталь уже сделана и зелёная — не трогаем, идём дальше.** Так весь продукт
постепенно заменяется рабочими кусками, ничего не ломая.

## Порядок слоёв (фазы LOVABLE-HANDOFF — строить ИМЕННО так, по очереди)
1. Shell + Chat (оболочка + ядро чата + macOS-док уже есть)
2. Modes (Мозг/Совет/Сравнение/Ресёрч/Дебаты)
3. Studio (картинки/видео/музыка/озвучка/3D)
4. Agent / Agent-OS
5. Memory / RAG / grounding
6. Code / Skills / MCP / терминал
7. Settings / кастомизация
8. Polish
Один слой полностью зелёный → следующий. НИКОГДА не начинать слой N+1 пока N не принят.

## АВТО-ПЕТЛЯ КАЖДОГО СЛОЯ (повторять до идеала — loop-until-perfect)
Для каждого слоя гнать этот цикл, пока два прогона подряд не выйдут чистыми:

```
ШАГ A — СТРОЙ (агент/воркеры)
  • fan-out: агент на каждую фичу слоя (dispatching-parallel-agents / Workflow)
  • ponytail full: минимальный код, без урезания валидации/безопасности
  • Sonnet на простую механику, Opus на сложную логику (opusplan)

ШАГ B — ВЕРИФИКАЦИЯ (на каждую фичу)
  • из taiga-web: npx tsc --noEmit  → 0 ошибок
  • трогал бэк: python3 -c "import ast;ast.parse(open('server.py').read())"
  • /app возвращает 200
  • смоук фичи: реально дёрнуть (клик/запрос), не «выглядит ок»
  • СКРИНШОТ фичи (Playwright / preview)

ШАГ C — СВЕРКА С ЭТАЛОНОМ (агент-критик)
  • визуал: сравнить скрин с design-v3 + docs/screenshots/ + usertest-jun15
  • поведение: сверить с FEATURE-INVENTORY (байт-в-байт, ничего не потеряно)
  • искать: мёртвые контролы, заглушки/TODO, регрессии, баги, расхождения дизайна

ШАГ D — ФИКС (если C нашёл ЧТО-УГОДНО)
  • fan-out фикс-агентов: агент на проблему
  • починили → ВЕРНУТЬСЯ к ШАГ B
  • два чистых прогона B+C подряд = слой ПРИНЯТ

ШАГ E — КОММИТ (только на зелёном)
  • сначала taiga-web, потом корень-указатель → push
  • один вынос = один коммит, никогда не коммитить сломанное
```

## A-to-Z ТЕСТ (после всех слоёв — полный прогон)
1. Прогнать ВСЕ юзер-кейсы из USER-CASES.md (каждый сценарий end-to-end).
2. Сверить визуал со всеми docs/screenshots/ (48) + usertest-jun15 (32 панели + 90 flows).
3. Прогнать TEST-CHECKLIST.md + QA-PLAN.md.
4. Финальный агент-критик «что упущено / не проверено / не идеально» → новый круг фиксов.
5. Security перед публичным выкатом: мина #0 (resolve_caller), мина #1 (RU-платёж).
6. Петля loop-until-dry: гнать, пока два полных прогона подряд не дадут НОЛЬ находок.

## Масштаб
ultracode, dynamic workflows, до 20 параллельных агентов. Память (graphify/gbrain/rag)
вместо догадок. Не уверен — спроси граф или Damir (RULE 0, не галлюцинировать).

## Можно как Workflow-скрипт
Этот поток ложится в Workflow-тулзу: pipeline(слои, строй→верифай→сверка→фикс),
parallel() для фичей слоя, loop-until-dry для петли. Опционально — закодить скриптом.
