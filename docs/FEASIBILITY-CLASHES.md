## КОНФЛИКТЫ между фичами (важно — реши до постройки)
**#1 🔴 КРИТИЧНО: «Без цензуры» ↔ RU-платёжки (YooKassa/СБП).** Российские эквайеры запрещают
uncensored/adult-контент в ToS — YooKassa может отказать/забанить мерчанта. Твой ГЛАВНЫЙ
дифференциатор (uncensored) конфликтует с RU-платёжными рельсами. РЕШЕНИЕ: крипта (BTCPay уже в
донорах) / зарубежный эквайер / нейтральная подача продукта платёжке. Это бизнес-блокер, не код.

**#2 🟡 Дорогие моды (Совет/Мозг/Сравнить/Дебаты, 2-5 моделей) ↔ дешевизна (ponytail) + «Save».**
Мульти-модель жжёт деньги, а цель — дёшево юзеру. Не противоречие, но нужен ЖЁСТКИЙ гард: бюджет-
слайдер (есть) + цена-перед-запуском (есть) + Save/кэш (планируется). Без гарда — юзер сожжёт баланс.

**#3 🟡 «Слияние 5→3 режимов (Мозг/Совет/Relay/Дебаты→Усилие)» ↔ решение «моды ОТДЕЛЬНЫ».**
Эта фича (FLEDGER:11, partial) = СТАРОЕ неверное понимание. Ты поправил: моды отдельны, усилие —
параметр. → ЭТУ ФИЧУ ВЫРЕЗАТЬ (она противоречит твоему решению).

**#4 🟢 Авто-роутер модели ↔ ручной выбор + Усилие.** Лёгкий клэш приоритета: ручной выбор должен
ПЕРЕБИВАТЬ авто.管ageable, просто правило precedence.

**#5 🟡 6 пересекающихся систем памяти:** vector-RAG (есть) + Mem0 + Letta + graphify + RAG-Anything
+ CLaRa + video-RAG + эпизодик. Строить ВСЕ = лишняя сложность/дубли. КОНСОЛИДИРОВАТЬ: один хороший
RAG (BM25+вектор, уже live) + опц. граф-память. НЕ тащить все 6.
