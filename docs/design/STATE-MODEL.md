# Тайга — модель состояния (per-chat vs аккаунт)

Сгенерировано мульти-агентным разбором (7 поверхностей → синтез → критик полноты), 2026-06-16.
Цель: что хранить на каком уровне, чтобы per-chat и аккаунт не путались.

## 6 уровней (сверху вниз, наследование account → … → message)

### 1. ACCOUNT — одно на весь аккаунт, синкается между устройствами
Вершина наследования + единая точка денег, приватности, безопасности.
- **Идентичность**: ID, имя, @handle, аватар, email/телефон, соц-входы, верификация
- **Безопасность входа**: пароль-хэш, 2FA/TOTP, резерв-коды, активные сессии/устройства, аудит-лог
- **Кошелёк**: баланс USD, история транзакций (pending/ok/failed/refunded/disputed) с idempotency-ключом, способы оплаты, промокоды/рефералы, автопополнение, возвраты/споры
- **Резерв средств (escrow)**: hold под фоновые/cron-прогоны; порядок резерв→факт→возврат; политика при нехватке денег
- **Подписка**: тариф, статус продления, лимиты/квоты, доступ к движкам по тиру
- **Модели**: дефолт-модель, избранные, дефолт-параметры генерации (вершина цепочки)
- **Промпты**: глобальные инструкции аккаунта (вершина цепочки)
- **Долгосрочная память**: память профиля «кто я», тумблер «Запоминать», эпизодическая кросс-чат память (читает только НЕ-изолированные чаты)
- **Знания (RAG)**: база знаний аккаунта, индекс/эмбеддинги, настройки RAG
- **Интеграции/секреты (vault)**: MCP-реестр, OAuth-токены (Fernet), ключи, BYOK — изолированный vault, redact в логах, НИКОГДА не в контекст модели/экспорт/публичную ссылку
- **Студия-дефолты**: движки, любимый голос, пресеты, общая библиотека ассетов
- **Биометрия-vault (отдельно)**: клон-голоса/образцы — отдельное согласие 152-ФЗ, свой TTL, запрет шаринга
- **Дизайн-дефолты**: бренд-система, тема канваса, фреймворк, политика песочницы превью
- **Агенты**: библиотека определений, маркет, шаблоны команд, дефолт-бюджеты/автономия, профиль доверия инструментов (allow/ask/deny), глобальная очередь фоновых задач
- **Scheduled jobs**: исполняемые экземпляры рутин/cron/лупов со снимком конфига, next_run, плательщиком, hold-бюджетом — переживают удаление/форк чата
- **Код**: killswitch «Режим разработчика», библиотека скиллов, MCP, хуки (+ неотключаемый safety-hook), лестница прав на тулзы
- **Комплаенс/uncensored**: юрисдикция, контент-фильтр как односторонний предохранитель ВСЕГО продукта
- **Оформление/локаль**: тема, акцент, язык, регион/TZ, шрифт/плотность/доступность
- **Приватность**: обучение на данных (opt-in/out), TTL истории/памяти, телеметрия, экспорт/удаление, версии Оферты, право-на-забвение
- **Синхронизация**: schema_version, разрешение конфликтов (LWW + entity-merge), оффлайн-outbox, квоты на объём состояния
- **Реестр публичных ссылок**: все ссылки чатов/артефактов с массовым отзывом
- **Организация-сервис**: дерево проектов, уведомления (email/push/Telegram), рефералы, бета-флаги

### 2. ORGANIZATION — опционально, только при шаринге (мультиплеер)
- ID команды, владелец, участники
- Общий кошелёк/квоты команды (чьи деньги тратит участник)
- ACL-роли: owner/editor/commenter/viewer; кто тратит/публикует; секреты не видит никто
- Правило «чьим ключом исполняется расшаренный агент»
- Аудит коллаборации

### 3. PROJECT — общее для папки чатов (средний слой наследования)
- Мастер-промпт проекта, общая память/RAG проекта, правила приватности источников
- Дефолт-модель проекта (override аккаунта), бюджет/лимит на проект
- Стиль/дизайн-система проекта, общая библиотека ассетов
- Рабочая директория проекта (общие файлы, куда пишут терминалы чатов)
- Закреплённые агенты/скиллы/MCP проекта, шаблоны рутин (определения)
- Закрепление чата в списке (свойство папки, не диалога)

### 4. CHAT — уникально для диалога (финальный override + единица приватности/экспорта)
- **Ядро**: история, заголовок, окно контекста, проект, шаринг/публичная ссылка, формат экспорта
- **chat.lastActiveMode** (Чат/Дизайн/Студия/Агент/Код) — persistent, для восстановления; политика переключения при незавершённом прогоне
- **Модель/генерация**: модель чата (override), режим ответа (Мозг/Совет/Дебаты), усилие, override-параметры сэмплинга
- **Промпт/память**: мастер-промпт чата, память чата (форк = снимок, НЕ утекает вверх/вбок), флаги наследования, флаг «изолированный чат» (вне кросс-чат памяти/экспорта/обучения)
- **Источники/инструменты**: включённые источники, активные коллекции, доступные MCP (override), прикреплённые файлы, живой документ
- **Студия**: override движка, библиотека ассетов чата, референсы, приватность ассета
- **Дизайн**: открытый артефакт + версии, токены-override, датасет графика, слайды, three.js, публикация
- **Агенты**: цель/роли команды, бюджет прогона, оркестратор, доска задач (канбан), история прогонов, определение лупа, уровень автономии
- **Код**: permission-mode чата (read-only→auto-edit→full), override скиллов/тулзов, история команд терминала

### 5. MODE-SESSION — эфемерно, в пределах одного прогона (клиентский draft, НЕ в БД)
- Активный режим текущей сессии, состояние стриминга (буфер, кнопка Стоп)
- Студия: режим медиа, активный пресет, очередь генераций (прогресс/ETA/отмена)
- Дизайн: несохранённый буфер, режим канваса (Превью/Код/Сплит), зум/выделение, орбита 3D
- Агенты: одноразовые «approve/reject», состояние прогона, итерация лупа, «allow once»
- Код: процесс run_code (subprocess, temp-cwd стирается)

### 6. MESSAGE — на один ход (иммутабельный снимок для воспроизводимости/биллинга)
- Ввод: черновик + вложения, веб-поиск (разовый тумблер), модель/режим на ход, override усилия, оценка стоимости ДО отправки
- Ответ: регенерация/ветка, цитаты, один tool_call/result (секреты замаскированы), мышление шага
- Статус: ok/error/canceled/timeout/filtered, списаны ли деньги за неудачу
- **Usage (факт)**: токены, реальная цена, чей ключ (наш/BYOK) — иммутабельно, отдельно от оценки
- Студия-параметры генерации (движок/seed/img2img/видео/музыка/голос/3D) — снимок + «повторить такое»

## Правая Рабочая панель = подписки на per-chat state
- **Предпросмотр** — живой рендер артефакта (зум/брейкпойнт эфемерны)
- **Картинки** — медиа Студии этого чата с метаданными генерации
- **Файлы** — файлы чата + вид на рабочую директорию ПРОЕКТА (общую)
- **Артефакты** — артефакты канваса + версии, живой документ, выходы агента
- **План** — цель/роли, канбан, оркестратор, определение лупа
- **Терминал** — история команд чата поверх каталога проекта (живой процесс эфемерен)
- **Фоновые задачи** — прогоны чата + проекция-ссылка в глобальную account-очередь (не дубль)
- **Память чата** — факты + тумблер «изолированный чат»

## Ключевое: одна цепочка наследования для «трёх близнецов»
**Промпт, Память, Модель-параметры** резолвятся одной функцией `resolve(key, chatId)`:
`account → (organization) → project → chat → (на ход) message`. Низ перебивает верх; пусто = берётся сверху. Возвращает значение + источник. В чате лежит ТОЛЬКО override (layered diff), не копия. UI: «по умолчанию из аккаунта» + «сбросить к наследуемому».

Жёсткие инварианты:
- Память чата НИКОГДА не утекает вверх/вбок; «изолированный» = вне кросс-чат памяти/экспорта/обучения.
- Тариф аккаунта — жёсткий потолок (замок в селекторе модели).
- Контент-фильтр account — предохранитель всего; ниже можно только УЖЕСТОЧАТЬ.
- Секрет никогда не в resolve()-контекст модели / экспорт / публичную ссылку / tool_call-лог.
- Форк чата: overrides и память — СНИМОК; cron/публичная ссылка/секреты НЕ форкаются.

## Зафиксированные решения Damir (2026-06-16)
- **BYOK убран.** Нет пользовательского vault чужих API-ключей. Все платят в кошелёк Тайги, Тайга платит провайдерам своими ключами. Один путь биллинга, проще. → из уровня ACCOUNT убрать «BYOK-ключи»; в MESSAGE usage всегда «наш ключ».
- **organization / team убран.** Продукт на одного пользователя: один аккаунт = один кошелёк. ACL-роли не строим сейчас. → удалить уровень organization и project.ACL; если позже «поделиться чатом» — добавим слой тогда.
- **Режим — свойство чата** (persistent `chat.lastActiveMode`), переключаемый; каждый режим имеет свой рабочий стол, правая Рабочая панель копит контент всех режимов чата.
- **Cron/автоматизации** — да: Рутины + Расписание (cron) + Фоновые задачи; на уровне ACCOUNT это scheduled jobs (исполняются в облаке по времени, переживают закрытие чата).

## Прочие открытые вопросы
1. **Режим — свойство чата или переключатель внутри?** → разбор: persistent `chat.lastActiveMode`, переключаемый; панель копит контент всех режимов чата.
2. **Изолированный чат** — бинарный флаг хватит, или нужен «помнить, но не в поиске»?
3. **BYOK**: на своём ключе — берём наценку с кошелька или бесплатно?
4. **Каталог ~800**: цены/доступность хранить локально (офлайн-замки/оценка) или дёргать у провайдера?
5. **Приоритет бюджетов** chat vs project vs org: жёсткий стоп на нижнем лимите или мягкое предупреждение?
6. **Расшаренный агент**: ключ владельца проекта или инициатора прогона?
7. **Биометрия 152-ФЗ**: TTL и форма согласия; отдельное удаление образцов?
8. **Cron ночью при нехватке баланса**: grace / пауза / отмена? Где частичный результат?
9. **organization сразу или MVP без шаринга?** Если мультиплеера нет — временно убрать роли из project.
