# Тайга — дорожная карта к проду (аудит 2026-06-16)

Сгенерировано мульти-агентным аудитом (10 агентов: инфра-карта по 8 способностям → когерентность фич → дорожная карта CTO).

## Вердикт (прямо)
**Это макет, не продукт.** `index.html` — чистый фронт: ноль fetch-вызовов, ни одного обращения к бэкенду. Красивая desktop-оболочка создаёт иллюзию готовности на 80%, но настоящие 80% работы (приём рублей под РФ, отказоустойчивый прокси к чужим API из РФ, модерация иллегала, антифрод на реальных деньгах) в макете почти не представлены.

**ЗАРАБОТАЕТ** — архитектура здравая, STATE-MODEL уже сделал половину работы стратега (BYOK убран, org/team убран → один путь биллинга кошелёк→ключи Тайги). Реальный MVP — это ~12-15 фич из ~95. Базовый чат с кошельком реально поднять за **2 недели**.

> Не строй desktop-OS из 95 фич на старте — построй один чат, который берёт деньги и отвечает, покажи живым людям, потом достраивай по спросу.

## С чего начать ПРЯМО сейчас (server.py, 2 коммита на этой неделе)
1. **КОММИТ 1 — безопасность денег (мина #0):** `resolve_caller(token)→uid` — резолвить пользователя из подписанного токена, а НЕ из поля `user` в теле запроса. Сейчас любой может подделать owner и тратить чужой кошелёк / получить RCE. Переключить `is_owner`/`require_balance` на него.
2. **КОММИТ 2 — приём рублей:** `billing.py` с интерфейсом `PaymentProvider` (create_invoice + verify_webhook(signature) + idempotent_credit), реализация ЮKassa; route `/api/pay/webhook` с HMAC-проверкой и идемпотентным зачислением; **удалить `test_topup=True` насовсем**; баланс → integer-копейки в SQLite.
3. **Параллельно:** вынести `gateway.py` — `chat(model, messages, params, stream)→нормализованный SSE` + `ping(model)` для liveness. Это убивает корневую причину «Failed to fetch» (мёртвые модели). **НЕ переписывать на фреймворк, оставаться на stdlib.**

## Безжалостный срез: 95 фич → ~15. Слить дубли:
- **Мозг/Совет/Сравнить/Relay/Дебаты/Глубоко/Ресёрч** → один селектор **«Усилие»** (Быстро/Подумать/Глубоко).
- **Веб-поиск/Супер-поиск/Проверка реальности/Источники правды** → один тумблер **«Поиск в вебе»** с источниками.
- **Память/Рабочая/Эпизодический/Экспорт** → одна **«Память»** (профиль + тумблер + изолированный чат).
- **RAG/Видео-RAG/Файлы/Проекты/Живой документ** → одна **«База знаний»**.
- **Прототип/Веб-артефакт/Слайды/Дизайн-система/Рисовалка** → один **«Холст»** (типы артефакта).
- **Рутины/Расписание/Доска/Прогоны** → одно **«Расписание»** (cron + статус).
- **Скиллы/Маркет/Конструкторы/Хуки** → одни **«Скиллы»** (конструкторы — фаза 2).

## Чего НЕТ, но обязательно для живого продукта
- **Онбординг + авторизация** (телефон/email + код, первое пополнение, «первый чат за 60 сек»).
- **Модерация иллегала** — односторонний предохранитель против CSAM/оружия/террора. Это про **уголовку фаундера**, не фича. Блокер запуска.
- **Платёжный контур РФ** — реальный приём рублей (ЮKassa/СБП), idempotency, возвраты/чарджбэки.
- **Лимиты + антифрод** — стоп при нуле баланса, rate-limit, защита от ботов. Иначе первый абузер уводит в кассовый разрыв.
- **Отказоустойчивость провайдеров** — таймауты, ретраи, фолбэк-бренд, НЕ списывать за неудачу.
- **Юробвязка** — Оферта, 152-ФЗ согласие, 18+ как флоу-блокер (не текст в футере).
- **Биллинг-прозрачность** — оценка стоимости ДО + факт-списание ПОСЛЕ.
- **Наблюдаемость** — логи, алёрт в Telegram при падении провайдера/всплеске расходов.

## Фазы (соло-фаундер, РФ)
- **Фаза 0 (нед. 1-2) — фундамент денег и доступа:** auth + кошелёк-ledger + ЮKassa + gateway.py + 6 захардкоженных моделей через зарубежный egress (DE/UK-серверы Mostik) + «первый чат за 60 сек» + списание usage (НЕ за неудачу).
- **Фаза 1 (нед. 3-4) — брать деньги и не разориться:** лимиты/антифрод + фолбэк-бренд/circuit-breaker/liveness + предохранитель иллегала + Оферта/ПДн/18+ + алёрты Telegram.
- **Фаза 2 (нед. 5-6) — показать людям:** Без цензуры + Поиск (SearXNG) + Усилие + Память + биллинг-UI + закрытая бета 20-50 РФ-юзеров.
- **Фаза 3 (нед. 7-10) — расширение по спросу:** База знаний (RAG) + Картинки (FLUX через релей, escrow) + TTS если просят.
- **Фаза 4+ (нед. 11+) — дорогое/опасное после выручки:** Агенты+cron (жёсткий escrow+kill-switch), Холст-песочница, Видео/музыка/3D поштучно, код-песочница.

## Главные риски
1. **Эквайринг-блок** — РФ-процессор может отказать из-за uncensored-позиционирования. Митигейт: нейтральное юр-описание («доступ к ПО/вычислениям») + второй процессор + СБП/крипта за тем же интерфейсом.
2. **Доступность провайдеров из РФ** — каталог в 700 моделей бесполезен, если живых из РФ единицы. Митигейт: liveness + circuit-breaker + фолбэк-бренд + egress на серверах Mostik (условие запуска).
3. **Деньги + асинхрон без человека** — cron может зациклиться/удвоиться/списать с пустого. Поэтому агенты-cron строго фаза 4, с escrow+идемпотентностью+kill-switch.
4. **Уголовка за иллегал** — без предохранителя первый абуз = статья на Дамира лично. Блокер запуска.
5. **Иллюзия готовности** — красивый макет соблазняет полировать UI вместо вайринга платежей. Держать линию 12-15 фич MVP.
6. **152-ФЗ/трансграничка** — ПДн россиян в РФ-БД, контент-исполнение за рубежом обезличенно + оферта про трансгранич.
