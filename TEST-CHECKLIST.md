# ТАЙГА ИИ — ПОЛНЫЙ РУЧНОЙ ЧЕК-ЛИСТ (current, после 17 батчей)

Прокликай это сверху вниз — ~1–2 часа на весь продукт. Помечай `[x]` что прошло, `[!]` что сломано.

**Окружение:** фронт `http://localhost:3000` (dev) → бэкенд `http://127.0.0.1:8777`.
**Легенда:** 🔑 = только владелец · 💸 = тратит деньги (реальный провайдер) · 🆓 = бесплатный путь · ⚠️ = известный нюанс/гэп (см. раздел в конце).
**Главные оси:** наверху 3 ЯРУСА — **Обычный / Студия / Ультра**. Внутри яруса — ось мощности **Fast / Auto / Expert / Heavy**.

Перед началом: открой DevTools → Network, чтобы видеть, какой `/api/...` дёргается. Один раз залогинься/открой как владелец (иначе 🔑-пункты будут скрыты — это правильно).

## 1. Ядро чата (отправка, стрим, стоп, повтор, правка, форк, копия, ветка)

- [ ] **Отправка** · напиши «привет», Enter (или ↑-кнопка) · → появляется твоё сообщение + ответ ассистента стримится посимвольно.
- [ ] **Shift+Enter** · перенос строки в композере, НЕ отправка.
- [ ] **Стрим виден** · во время ответа текст растёт постепенно, не появляется целиком.
- [ ] **Стоп** · нажми ■ (Square) во время ответа · → генерация обрывается, частичный текст остаётся, кнопка снова становится ↑.
- [ ] **Повтор / regenerate** · под ответом нажми ↻ (RotateCcw) · → тот же вопрос гонится заново, новый ответ.
- [ ] **Правка своего сообщения** · наведи на своё сообщение → карандаш (Pencil) → измени текст → отправь · → ветка перегенерируется с правкой.
- [ ] **Копировать** · кнопка Copy под ответом · → текст в буфере (вставь куда-нибудь, проверь).
- [ ] **Форк диалога** `/branch` (GitFork) · → текущий диалог копируется в НОВЫЙ чат, оригинал цел (проверь в сайдбаре — два чата).
- [ ] **Дерево веток** · после правки/повтора под сообщением появляются стрелки `‹ 1/2 ›` для переключения вариантов ответа.
- [ ] **Очистить** `/clear` · → диалог пуст, начинается заново.
- [ ] **Сохранение при перезагрузке** ⚠️ · напиши пару сообщений → F5 · → проверь, что чат не пропал (это исторически слабое место — см. KNOWN GAPS CB-08).
- [ ] **Несколько чатов** · создай новый чат (+ в сайдбаре), переключайся между чатами · → история у каждого своя.
- [ ] **Переименование / удаление чата** · в сайдбаре правый клик/⋯ по чату · → переименуй, удали.

## 2. Режимы мощности и ярусы (Fast / Auto / Expert / Heavy · Обычный / Студия / Ультра)

- [ ] **Ярус Обычный** · верхняя навигация → «Обычный» · → обычный чат, 700+ моделей.
- [ ] **Ярус Студия** · «Студия» · → открывается медиа-студия (картинки/видео/музыка/3D) — см. раздел 5.
- [ ] **Ярус Ультра** · «Ультра» (золотая подсветка) · → топ-модель + глубина + без отказов.
- [ ] **Fast** (Zap) · быстрая не-думающая модель, коротко · задай простой вопрос → ответ почти мгновенно.
- [ ] **Auto** (Wand2, дефолт) · авто-роутер сам выбирает модель под задачу.
- [ ] **Expert** (Brain) · думающая модель, разбор по шагам → развёрнутый ответ (дольше Fast — это норма).
- [ ] **Heavy** (Network) · «совет/мозг»: дешёвая-контролёр рулит, сильную зовёт на сложное · ответ заметно медленнее, может показать шаги «спрашиваю эксперта».
- [ ] ⚠️ **Fast + reasoning-модель = медленно** · вручную выбери думающую модель (`/model`) и оставь Fast · → ответ будет МЕДЛЕННЫЙ, т.к. модель всё равно «думает». Fast НЕ фильтрует reasoning-модели — это известный гэп, не баг отправки.
- [ ] **Переключение оси на лету** · в одном чате поменяй Fast→Expert→Heavy между сообщениями · → каждое сообщение уважает выбранную ось.

## 3. Мульти-модель (Brain · Relay · Council · Compare · Research)

- [ ] **Brain / Мозг** (через Heavy или пилюлю) · сложный вопрос · → дешёвая модель ведёт, зовёт сильную; показывает шаги (если включено). ⚠️ Медленнее обычного — два прохода.
- [ ] **Настройки Мозга** (шестерёнка у пилюли) · смени ведущего (lead) и эксперта (expert вручную vs auto), вкл/выкл «показывать шаги» · → настройки сохраняются (F5, проверь).
- [ ] **Relay / Связка** · вкл Relay · → дешёвая-без-цензуры причёсывает запрос → умная отвечает (один финальный ответ). ⚠️ Два прохода → медленнее.
- [ ] **Council / Совет** (Ультра → пилюля «Совет» или `/council`) · → несколько моделей отвечают → синтез в ОДИН ответ. 🐢 Медленно (N моделей + синтез).
- [ ] **Выбор моделей Совета** · открой мульти-селект, выбери 2–5 моделей + модель-синтез · → именно они участвуют.
- [ ] **Compare / Сравнение** `/compare` · → ответы выбранных моделей БОК-О-БОК, без синтеза (карточки рядом). 🐢 Медленно (фан-аут).
- [ ] **Research / Ресёрч** `/research` или пилюля · → глубокий ответ с источниками `[1][2]`. 🐢🐢 Самый медленный (поиск по источникам + синтез). Проверь глубину (fast/medium/deep) и число источников в шестерёнке.
- [ ] **Beam-fusion** · технически это Совет с `beam=True` (де-галлюцинированный фан-аут→мердж) — если есть переключатель, проверь, что выдаёт один «очищенный» ответ. 🐢 Медленно.
- [ ] **Взаимоисключения** · Fast / 2-модельные пути / картинка отключают council/research — проверь, что UI не даёт включить несовместимое одновременно.

## 4. Модели (пикер, категории, поиск, BYOK, экран каталога, обновление)

- [ ] **Пикер модели** `/model` · → список 700+ моделей с категориями (чат/код/зрение/без цензуры/…).
- [ ] **Поиск в пикере** · введи «opus»/«deepseek»/«qwen» · → фильтрует по имени.
- [ ] **Категории/фильтры** · фильтры по контексту / свободе / цене / языку / специальности.
- [ ] **Бейджи** · в каталоге видны бейджи: новое / самое дешёвое / TEE / % без цензуры.
- [ ] **Экран каталога** (Настройки → «Каталог моделей» или модалка) · → полноценный экран с глубокими фильтрами + сохранёнными чипами-фильтрами.
- [ ] **Клик по бренду** · нажми бренд (напр. Anthropic) · → фильтр на самые свежие модели бренда.
- [ ] **Refresh каталога** · кнопка RefreshCw / `/api/catalog/refresh` · → каталог пересобирается без рестарта (баланс/новые модели подтягиваются).
- [ ] **BYOK 🔑/настройки** · Настройки → Интеграции → ключи Venice/NanoGPT/Chutes/Redpill(TEE) · вставь свой ключ → его модели подмешиваются; убери → исчезают.
- [ ] **Health-бейджи** ⚠️ · если включена индикация up/down — мёртвые провайдеры скрываются/помечаются (batch 21 частично; может быть базовым).

## 5. Студия (картинка, img2img, видео, музыка, 3D, TTS, аватар, кино, код-рендер, param-form)

- [ ] **Открыть Студию** · ярус «Студия» · → вкладки Картинки / Видео / Музыка / 3D / Аудио / Кино.
- [ ] **Картинка 💸/🆓** · «нарисуй кота» → Генерировать · → реальная PNG. Цена показана ДО генерации.
- [ ] **img2img / правка** · загрузи init-изображение (Upload) → промпт → генерация на его основе.
- [ ] **negative_prompt** · заполни поле «избегать» · → реально уходит как negative_prompt (batch 10).
- [ ] **Галерея** · сгенерённые картинки копятся в галерее, можно скачать (Download).
- [ ] **Видео 💸** ⚠️ · t2v / i2v · → submit→poll→прогресс→ролик. Может быть 402 (кошелёк AIMLAPI пуст) — тогда дружелюбная ошибка, НЕ краш (см. KNOWN GAPS VID-01/02).
- [ ] **Музыка 💸/🆓** · промпт → submit→poll→COMPLETED → `<audio>` играет mp3, можно скачать.
- [ ] **3D** ⚠️ · фото → 3D · → backend `/api/td3` fast-fail (≤13с, без зависа на 30с). `.glb` провайдер может быть down — это известно.
- [ ] **TTS / Аудио 💸** ⚠️ · текст → озвучка (Kokoro/OpenAI, голос af_bella) · → mp3. Может быть 402 (кошелёк NanoGPT TTS пуст) — дружелюбная ошибка.
- [ ] **Аватар (говорящая голова)** · в Кино-студии тип сцены «avatar» · → лицо + озвучка.
- [ ] **Кино-студия** · собери сториборд из сцен (видео/картинка/оживить/аватар), у каждой своя модель/голос/ракурс · → «Склейка» → `/api/cinema-export` (ffmpeg) → MP4. (SSRF-гард на склейке.)
- [ ] **Код-рендер 🆓** · в Студии «🆓 код-рендер» для диаграмм/постеров/схем · → рендерится кодом, без денег. Проверь диаграмму, SVG, график, слайды.
- [ ] **Param-form** ⚠️ · у моделей с inputs-схемой форма параметров (длительность/fps/разрешение через слайдеры/селекты). Базовые параметры работают; продвинутые могут быть косметикой (см. KNOWN GAPS).

## 9. MCP (подключение без auth, token-auth, ресурсы, инструменты в чате, тумблер)

- [ ] **Панель MCP** `/connect` (Plug) 🔑 · → маркет коннекторов (Notion/GitHub/своя БД) + своё по URL.
- [ ] **Подключение без auth** · коннектор без токена → «Подключить» · → появляется в списке.
- [ ] **Token-auth 🔑** · коннектор с токеном → вставь токен → подключить · → токен шифруется (Fernet) на бэке.
- [ ] **Тумблер вкл/выкл** · Power-кнопка на коннекторе · → включает/выключает его инструменты.
- [ ] **Ресурсы** · у коннектора видны resources/prompts (FileText).
- [ ] **Инструменты в чате** · с включённым MCP и агентом — модель вызывает инструменты коннектора в обычном чате.
- [ ] **Своё по URL 🔑** · добавь свой MCP по URL · → SSRF-гард; битый URL → `{ok:false,error}` (UX корректный, см. KNOWN GAPS — 200 вместо 400, косметика).

## 10. Память и RAG (профиль, факты, сленг, удаление-надгробие, sleep-time, ingest/query, smart-RAG, мультимодал, пер-чат)

- [ ] **Память чата** `/memory` (Database) · → что Тайга помнит в ЭТОМ чате, правится руками.
- [ ] **Забыть чат** `/forget` · → стирает память текущего чата.
- [ ] **Профиль-факты (между чатами)** · панель Memories · факты авто-копятся; добавь свой, отредактируй, удали.
- [ ] **Tombstone-удаление** · удали факт · → авто-память НЕ вернёт удалённое (надгробие) — задай вопрос, провоцирующий тот факт, проверь, что не всплыл.
- [ ] **Заметка «как ты пишешь» (сленг/стиль)** · в Memories есть стиль-заметка; пиши со сленгом/T9 → `/api/extract_style` обновляет; ответы подстраивают тон.
- [ ] **Применение профиля** · профиль/стиль применяются к НОВЫМ сообщениям после перезагрузки (открытие чата подгружает их).
- [ ] **Sleep-time консолидация 🔑** · `/api/memory_consolidate` (ручной триггер владельца) или фон при простое · → дубли памяти мёржатся/чистятся (batch 15).
- [ ] **RAG ingest** · `/knowledge` (Library) → загрузи документ · → `/api/rag_ingest` → эмбеддинги (структурно-осознанный чанкинг).
- [ ] **RAG query** · задай вопрос по загруженному доку · → семантический поиск находит релевантное.
- [ ] **RAG delete** · удали документ из базы знаний · → больше не находится.
- [ ] **Smart-RAG** · вкл тумблер «умный поиск» в композере · → multi-query + RRF hybrid + LLM rerank (медленнее, точнее). Выкл → одиночный запрос (back-compat).
- [ ] **Мультимодальный RAG** · загрузи документ с картинками/таблицами или скан-PDF · → VLM подписывает изображения, текст подписей индексируется (batch 13).
- [ ] **Пер-чат workspace** ⚠️ · RAG можно тегать по chat_id (AnythingLLM). НО основной путь чата по умолчанию ищет по ВСЕМ докам — «query within this chat» пока не передаётся из главного чата (back-compat, известный follow-up).
- [ ] **Эпизодическая память** `/recall` · → поиск по прошлым чатам (`/api/recall`).

## 17. Обработка ошибок и устойчивость

- [ ] **Пустое сообщение** · нажми отправить с пустым полем · → ничего не отправляется.
- [ ] **Огромный ввод** · вставь очень длинный текст · → не вешает UI; либо обрезка/сжатие, либо корректная отправка.

- [ ] **Heuristic tool-parser** · на не-function-calling модели (uncensored) попроси действие с инструментом · → парсер вытаскивает вызов из текста; прозо-JSON НЕ ломает (фикс был).

## KNOWN GAPS (не пугайся — это уже задокументировано прошлыми ревью)

- **/sprint как «мёртвая кнопка»** · команда `/sprint` РАБОТАЕТ (owner self-test). Если в каком-то меню остался декоративный «спринт/sprint» пункт без эффекта — это пережиток, не функциональный баг.
- **Orphan `playground.tsx`** · компонент per-model playground (`components/playground.tsx`) собран, но НИГДЕ не подмонтирован (нет импорта в settings/chat/app). Фича существует в коде, но недостижима из UI — ожидаемо.
- **Студийные «продвинутые параметры» косметика** · param-form рендерит базовые контролы; часть продвинутых полей у некоторых моделей — пока косметические (не все доходят до провайдера).
- **Permission-UI (batch 12) дремлет** · типизированные SSE-события + интерактивная модалка прав (allow once/always/deny) реализованы, но back-compat: если клиент не «договаривается» о permission-wait, бэкенд НЕ ждёт. На большинстве путей модалка прав может не всплывать — это намеренная обратная совместимость, не поломка.
- **Reasoning-модели не фильтруются в Fast** · Fast-режим не отсекает думающие модели; вручную выбранная reasoning-модель в Fast будет МЕДЛЕННОЙ. Известный гэп, не баг.
- **Пер-чат RAG не передаётся из главного чата** · workspace-scope есть в бэке и билдерах, но основной send-путь не шлёт chat_id → RAG по умолчанию ищет по ВСЕМ докам (back-compat). Follow-up.
- **MCP add по URL возвращает 200 на SSRF-отказе** · тело несёт `{ok:false,error}` (фронт читает корректно, UX в порядке), но HTTP-код несогласован с `action:"add"` (400). Косметика.
- **Видео / TTS могут быть 402** · `VID-01/VID-02/VO-05` — провайдерские кошельки (AIMLAPI video, NanoGPT TTS) бывают пусты. Тогда дружелюбная ошибка — это не дефект кода, а top-up кошелька.
- **`.glb` 3D-вывод** · провайдер triposr/AIMLAPI бывает down — известно, не чинится кодом.
- **Сохранение чата при F5 (CB-08)** · исторически слабое место персистенса — проверь специально (раздел 1).
- **Дублирование `McpConnectSection`** · в agent-builder и skill-builder — DRY-долг, не влияет на работу.
- **Отложено (нужны ключи/инфра владельца):** реальные платежи (СБП/BTCPay/crypto), self-hosted голос/guard, sqlite-vec, E2B-песочница, ComfyUI/manim/FLUX-schnell, Taiga-Coder v2 автономный цикл, GPU-хост, desktop-приложение, соц-публикация, UI логина, ACE-Step музыка, deepfake/face consent-гард, MCP OAuth full-flow, git-push/PR коммиты агента.
