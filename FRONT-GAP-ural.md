# Карта «Фронт vs Движок» — что юзер видит, чего нет (2026-06-12, ural)

39 эндпоинтов бэка × фронт taiga-web. Архитектура: каждый `app/api/<x>/route.ts` — простой
прокси на 127.0.0.1:8777. Фронт-лёгкий: бэк держит тяжёлое, UI = показ + команды.

## 🟢 РАБОТАЕТ У ЮЗЕРА (18 — ядро продукта готово)
chat · image · video · audio · cinema · супер-поиск (`/search`) · веб-поиск (`/web`) ·
браузер (browser-panel) · агенты-оркестратор (`/agents`) · RAG ingest+query (sources-panel) ·
MCP (`/connect`) · каталог моделей (model-picker) · баланс (показ) · извлечение файлов (PDF/DOCX) ·
compact · авто-память (mem_extract) · улучшение промпта (`/improve`).
→ **Главное: продукт юзабелен прямо сейчас для одного юзера.**

## 🟡 РАБОТАЕТ, НО ЛОКАЛЬНО / ИЛИ ТОЛЬКО OWNER (норм для запуска)
- память recall/forget · скиллы · настройки → живут в localStorage (юзер фичу имеет, просто
  не синкается на сервер; для старта ок)
- catalog_refresh · billing · users · apikeys · identity → owner/админ, юзер-UI не нужен
- websearch → уже есть через флаг `/web` в чате (отдельный эндпоинт не нужен)

## ✅ ЗАКРЫТО ural'ом (2026-06-12, фронт-подключение готовых фич)
Юзер теперь может пользоваться через команды + панели с окнами ввода:
- **/cron** — агенты по расписанию (jobs-panel): окно «задача + интервал», список, пауза, удаление.
- **/cookies** — вход по cookies (cookies-panel): окно вставки Netscape/JSON, список, удаление.
- **/knowledge** — база знаний RAG (knowledge-panel): список документов + удаление.
Сделано без клэша: 3 новых прокси-роута + 3 хелпера + 3 панели (новые файлы) + хирургический
монтаж в chat.tsx/builtins/command-palette под замками. tsc=0. Прокси проверены вживую через
dev-сервер :3000 (браузер→прокси→бэк→ответ для всех трёх). Бэк-путь jobs/cookies/rag проверен.

## 🔴 РЕАЛЬНЫЕ ДЫРЫ ДО ПУБЛИЧНОГО ЗАПУСКА (нужен фронт CORE/amur)
| Дыра | Что построено в бэке | Чего нет на фронте | Блокер для |
|---|---|---|---|
| **Вход/аккаунты** | `/api/auth` (signup/login, токены) | нет экрана логина/регистрации (в builtins «soon») | мультиюзер-запуска |
| **Оплата** | `/api/topup` (гейт 503 до процессинга) | нет экрана пополнения/оплаты | монетизации (Damir: «later») |
| **Cron-агенты** | `/api/jobs` (планировщик) | нет UI расписания/задач | power-фичи (не блокер) |
| **Cookies-вход** | `/api/cookies` (шифр-хранилище) | нет UI вставки cookies | co-browse логина (Damir: «test later») |
| **Чистка RAG** | `/api/rag_delete` | нет кнопки удалить документ | гигиены (мелочь) |
| **История чатов** | `/api/chats` `/api/save` `/api/delete` | нет управления списком/сохранением на сервере | кросс-устройство (локально работает) |

## ВЕРДИКТ
- **Ядро (чат + вся генерация + агенты + поиск + RAG + браузер) — подключено и работает.**
- До запуска на ОДНОГО (тебя) — по сути готово.
- До ПУБЛИЧНОГО мультиюзер-запуска не хватает двух вещей: **экран входа** и **экран оплаты**.
  Обе — фронт-лана (amur/kedr); бэк под них готов (auth.py + topup-гейт ждут).
- Остальное (cron-UI, cookies-UI, rag-delete) — кнопки к готовым фичам, не блокеры.

Бэк под все красные дыры уже стоит — нужен только фронт. Контракты: BACKEND-API.md.
