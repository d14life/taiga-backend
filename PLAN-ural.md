# PLAN — ural (BACKEND)

Принцип: **depth, not breadth**. Каждая задача — до конца + тест (py_compile → restart → curl).
Порядок по «заплатит ли юзер ИМЕННО за это». Лана: server.py(ядро)/orchestrator/browser_hub/guard.
Общие файлы (select.ts) — через замок. api_chat/каталог/поиск/браузер/rag/оркестратор = мои.

## Очередь (depth-first)
1. **RAG в чате** — авто-инъекция retrieved-контекста в api_chat (если у юзера есть доки).
   Без этого RAG-endpoint бесполезен. Тест: ingest doc → chat → grounded-ответ со ссылкой.
2. **Оркестратор SSE-стрим** — `/api/orchestrate` стримит timeline (plan/agent/synth) живьём,
   а не финальный массив. Для панели как Copilot Agents. Тест: curl -N, видеть события.
3. **Оркестратор: последовательный/DAG режим** + super_search как инструмент воркера
   (researcher-воркер реально ищет). Тест: зависимая задача «найди X → раскритикуй».
4. **Браузер idle-очистка** — закрывать неактивные сессии (>10мин), лимит N сессий. Тест: открыть, ждать, проверить очистку.
5. **Decision-gate «Решает сам»** backend в api_chat — риск-скоринг (порт OpenHands) +
   SSE `decision` event на рискованных действиях. Тест: триггер → событие.
6. **Каталог авто-рефреш** раз в 6ч (фоновый поток) — новинки без рестарта. Тест: проверить тред.
7. **Permission-ladder** — гейт серверных тулзов по тарифу. Тест: low-tier → отказ на dev-тулзе.
8. **Cookies шифрохранилище** + **ECC-скиллы** в оркестратор (расширить персоны).

## Правила
- Никогда не оставлять сломанный билд (py_compile + рабочий рестарт после каждой задачи).
- test_topup остаётся CLOSED. Секреты редактировать. Без push/удаления/оплаты.
- Блок на решении → QUESTIONS-ural.md, безопасный дефолт, ИДУ дальше.
