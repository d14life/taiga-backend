# Тайга ИИ — большой аудит системы (для понимания, фронтир-уровень)

Документ объясняет ПРОСТЫМ языком: как Тайга выбирает модель, что за каталог под капотом,
как работает деградация по уровням усилия, и какие есть режимы/функции с их настройками.
Дата: 2026-06-15.

---

## 1. Как Тайга выбирает модель (роутер)

Принцип: **«каталог = роутер»**. Не хардкод «провайдер X для задачи Y», а живой каталог моделей
с полями (цена, ум, размер, цензура, приватность, контекст), по которому выбирается лучшая.

### Когда модель падает или провайдер пуст — порядок замены (до 4 альтернатив):
```
1. ТА ЖЕ модель у другого ОПЛАЧЕННОГО провайдера   (точная копия)
2. Тот же КЛАСС (вендор+размер), БЛИЖАЙШАЯ версия    (opus-4.8 → 4.7 → 4.6)
3. Страховка для uncensored: оплаченный uncensored близкого ума
   (НИКОГДА не прыгаем в дорогую цензурную модель, если ты выбрал uncensored)
```
Внутри каждого уровня сортировка: совпало uncensored → **дешевле (цена за 1к сообщений)** →
**полнее кошелёк провайдера (баланс)**. Дорогой/чужой вариант (напр. не тот Opus) — в самый конец.

**Учитывает то, что ты просил:** остаток баланса каждого аккаунта, цену каждой модели, до 4 запасных,
ту же модель у других провайдеров, ближайшую семью/версию.

### Защита твоего выбора
- Выбрал модель ЯВНО — система её НЕ меняет, пока она жива. Свап только если она реально мертва.
- Бюджет тесный — сначала ↓глубину размышления (твоя модель цела), смена модели только в авто-режиме.
- Провайдер в 402/просел — `_first_live`/роутер обходят его и берут живую модель у оплаченного
  (это чинило висяк дизайн-канваса: раньше no-model запрос садился на мёртвый Venice).

## 2. Каталог моделей — живая авто-обновляемая база

| Что | Значение |
|---|---|
| Моделей сейчас | ~990 (текст+медиа) |
| Провайдеров | 5: NanoGPT, Venice, Redpill, Chutes, **AIMLAPI** (твои $20 теперь в деле) |
| Откуда | живой фетч `/models` каждого провайдера (цены, контекст, флаги) |
| Обновление | **фоновый пересбор каждые 6 часов** + по кнопке «Обновить» — новинки без рестарта |
| Кэш | `rich_catalog.json` на диск (переживает рестарт; провайдер-флап не вырезает модели) |
| Дедуп | одна модель у нескольких провайдеров → оставляем самую ДЕШЁВУЮ, остальные в «также» |

**Вывод:** это не статичный список — это самообновляющаяся база ровно тех моделей, которые ты
реально можешь вызвать (по твоим оплаченным ключам). «Полнее» базы для роутинга быть не может —
лишние модели, которые нельзя вызвать, только мешали бы.

## 3. Пять уровней усилия (деградация мышления)

Ось интеллекта — отдельная от длины ответа и от режима. Слайдер «Глубина», 5 уровней + **Авто**:

| Ур. | Метка | Нативный движок размышления | Когда |
|---|---|---|---|
| 1 | Быстро | reasoning OFF | привет/спасибо/тривиальное |
| 2 | Лайт | reasoning LOW | короткий простой вопрос |
| 3 | Норма | reasoning MEDIUM (дефолт) | обычный/пояснительный |
| 4 | Глубже | reasoning HIGH (+ thinking-версия модели) | код/докажи/отладь |
| 5 | Макс | HIGH + видимый разбор по шагам | очень длинный / «подробно» |
| — | **Авто** | классифицирует сложность вопроса → ставит 1–5 сам (мгновенно, без вызова ИИ) | дефолт |

**Деградация по бюджету:** при тесном балансе система сначала снижает уровень усилия (дешевле, но
ТВОЯ модель цела), и только в авто-режиме может сменить модель на более дешёвую того же класса.

## 4. Режимы ответа (что выбрать под задачу)

| Режим | Что делает простым языком | Когда брать | Настройки юзера |
|---|---|---|---|
| **Чат** | Одна модель отвечает напрямую | по умолчанию | глубина, модель, длина |
| **Мозг** | Дешёвый «ведущий» триажит вопрос и зовёт сильного эксперта только когда реально надо | трудные вопросы без лишних трат | число экспертов 1–3, ведущий, эксперт |
| **Совет** | 2–5 моделей отвечают независимо → одна сводит в общий ответ | важное решение, нужен консенсус | сколько моделей (2–5), какие, кто сводит |
| **Слияние** | Как Совет, но сведение через критика-фьюжн (режет галлюцинации) | когда важна точность фактов | то же, что Совет |
| **Сравнение** | 2–5 моделей бок-о-бок, БЕЗ сведения — сам смотришь | хочешь увидеть разные мнения | какие модели (2–5) |
| **Ресёрч** | План → поиск по движкам (Sonar/Exa/Brave/Gemini) → отчёт с цитатами [N] | глубокая тема, нужны источники | глубина (быстро/сбаланс/глубоко), число источников |
| **Связка** | Дешёвая модель переписывает твой запрос почётче → сильная отвечает | мутный/короткий запрос | инструкция переписи, ответчик |
| **Агент** | Модель сама зовёт инструменты (поиск, код, картинки, браузер) под цель | многошаговая задача с действиями | права план/авто/полный, аппрув, интерактив |
| **Агент-ОС** | Весь чат через харнес: цель → разбор → действие → проверка → повтор | сложная автономная работа | ценовой тир, события-таймлайн |
| **Источник правды** | Отвечает ТОЛЬКО по твоим загруженным источникам, факты вне их помечает | юр/факт-чек по своим докам | вкл + загрузить доки |
| **Характер** | Персона (Архитектор/Критик/Писатель/Ресёрчер/Наставник/свои) | задать тон/роль | выбор персоны; анти-сползание держит её |
| **Пресеты** | Режим+модель+промпт+флаги одной кнопкой (Юрист/Кодер/…) | повторяемые сценарии | свои пресеты (до 50) |

## 5. Контролы композера, инструменты и параметры

### Контролы у поля ввода (сгруппировано)
- **Ввод:** файлы/фото (PDF, DOCX, изображения…), скриншот экрана, камера, голос, hands-free.
- **Интеллект:** слайдер глубины 1–5 + **Авто**, пикер модели (Авто/ручной), авто-режим, «улучшить промпт».
- **Бюджет/масштаб:** длина ответа (эконом/норма/макс), ценовой тир (любая/дёшево/средне/топ),
  лимит трат ($0/0.05/0.2/1), окно контекста (0=всё … 40 сообщений).
- **Поведение:** характер, анти-сползание, «уточнять» (спросит вместо догадки), приватно (TEE),
  призрак (чат не сохраняется), связка, мозг.
- **Агент:** права (план/авто/полный), политика аппрува (спрашивать/умный/полный), интерактивный
  агент, сессия-агент.
- **Быстрый доступ:** пресет-чипы, закреплённые команды, подсказки «что дальше», морф-чип
  (предложит «запустить как цикл/команду»), меню «+» (~30 функций).

### Инструменты, которые может звать модель
- **Безопасные (всем):** веб-поиск, супер-поиск (Sonar+Exa+Brave+Venice), fetch URL, Википедия,
  курсы валют, калькулятор, дата/время, поиск по навыкам, генерация картинки, запомнить факт.
- **Dev (только владелец/полные права):** shell, запуск кода (песочница), чтение/запись/правка
  файлов. Деструктив (`rm -rf /`, `mkfs`, …) блокируется на уровне хука.
- **Гейт прав:** план → блок мутаций; авто → безопасные сразу, рисковые по подтверждению; полный → всё.

### Главные параметры запроса (для тонкой настройки)
`model` (или «Авто») · `reasoning_effort` low/medium/high · `tier` cheap/mid/top · `max_tokens` ·
`temperature` · `system` · `brain`+`brainExperts` · `relay` · `research`+`depth`+`sources` ·
`council`/`compare`/`beam`+`n` · `agent`/`agent_os`+`perm` · `grounded` · окно памяти · авто-проверка.

## 6. Находки и рекомендации (как у фронтир-аудитов)

**Хорошо (подтверждено кодом):**
- Роутер реально балансо-/цена-/класс-осознанный: та же модель → ближайшая версия → семья → дешевле+полнее-кошелёк. Опус только в крайнем случае.
- Каталог — живая авто-обновляемая база (5 провайдеров, 6ч рефреш, дедуп по цене).
- 5-уровневая деградация усилия консистентна; бюджет сначала жмёт глубину, не модель.
- Авто-проверка ответа (галлюцинации/деградация) + бейдж уверенности.

**Найденные дыры (часть уже починена в этой серии):**
1. ✅ ИСПРАВЛЕНО: no-model запросы (дизайн-канвас) садились на мёртвый Venice → висяк. `_first_live` теперь обходит просевших провайдеров.
2. ✅ ИСПРАВЛЕНО: AIMLAPI ($20) был невидим для текста → подключён 5-м провайдером (+UA-фикс 403).
3. ✅ ИСПРАВЛЕНО: фолбэк не знал класс/версию → прыгал в Opus. Теперь та же/ближайшая версия.
4. ⚠️ AIMLAPI: цены по моделям их API не отдаёт → стоит консервативная оценка. Следить за тратами $20; при перерасходе поднять цену, чтобы aimlapi всплывал только для уникальных моделей.
5. ⚠️ Дефолт-цепочка роли «chat» — вся на Venice. Сейчас `_first_live` обходит это, но саму цепочку стоит диверсифицировать (добавить nano/chutes-модели), чтобы не зависеть от обхода.
6. 💡 Возможность: вынести 5-уровневое усилие в per-режим (у Совета/Ресёрча своё), как ты и предлагал — «деградация по 5 уровням для других режимов/функций».
