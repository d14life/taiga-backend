# PLAN — kedr (CORE)

Принцип: глубина, не ширина. Каждая фича — РЕАЛЬНАЯ и полная (прокси + UI + проверка в браузере),
ноль заглушек. Бэкенд готов (ural/atlas) — я ПРОВОЖУ + ПОЛИРУЮ.

Порядок (по «заплатит ли юзер ИЗ-ЗА этого», + быстрый ранний win):

1. **₽ + низкий баланс красным** (быстро, реальная польза, проверяет пайплайн прокси→UI)
   - /api/init balances[provider].low → провайдер красным в каталоге/пикере
   - цена ответа ₽ виднее (уже есть курс) — где показываем «$», добавить ₽
2. **Супер-поиск: панель источников/цитат** (Perplexity-паритет — за это платят)
   - прокси /api/supersearch; в Ультре фан-аут по движкам → синтез + список источников с номерами
   - инлайн-цитаты [1][2] кликабельны → источник
3. **Браузер-в-чате панель** (co-browse — наш ров)
   - прокси /api/browser (open/act/close); панель: скриншот+текст страницы, юзер видит, ИИ кликает
4. **Оркестратор-таймлайн** (их клин «в Claude нет»)
   - прокси /api/orchestrate; панель: мозг→воркеры→синтез, живой таймлайн шагов

CORE-LATER (после 4): темы · пресеты · папки чатов.

## Правила
- Своя лана (chat/sidebar/settings/model-picker/canvas/master-prompt + src/lib/* кроме *-gen + api-прокси) — правлю свободно.
- ВНЕ ланы / общий (server.py, select.ts) — сперва `claim`, после — `release`.
- tsc=0 всегда. Браузер-проверка каждой фичи. Блок → QUESTIONS-kedr.md, безопасный дефолт, дальше.
