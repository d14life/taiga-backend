# ГРАНД-ПЛАН: МАКС-КАСТОМИЗАЦИЯ + РЕЖИМ РАЗРАБОТЧИКА (для юзера, безопасно)
2026-06-12 · Damir-directed: "allow developer mode for each user to change each mode, with security, reach only what they can without breaking security"

## ВИЗИЯ
Каждая функция = **кнопка-пилюля + своё окно настроек**, которое **юзер сам крутит**. Плюс «Режим разработчика» = максимум ручек на КАЖДЫЙ режим. Плюс юзер **собирает свои функции** из кирпичиков. ИИ строит конфиг по словам. ВСЁ в песочнице — юзер меняет ПОВЕДЕНИЕ, но никогда не получает доступ к коду/чужим данным.

## 🔐 МОДЕЛЬ БЕЗОПАСНОСТИ (жёстко — это и есть «security of course»)
**МОЖНО менять юзеру (whitelist, валидируется НА СЕРВЕРЕ):**
- модель на каждый режим · длину ответа (max_tokens) · «температуру»/креативность · системный промпт/персону НА режим · какие инструменты включены (web/calc/…) · окно памяти · формат ответа · собрать свою функцию из существующих примитивов · свой ключ провайдера (BYOK).

**НЕЛЬЗЯ (жёсткая стена, даже если попросит):**
- owner-инструменты (shell/чтение-запись файлов/код-исполнение) · чужие данные · системные ключи · серверный код/внутренности · правка баланса/биллинга · owner-гейтнутые роуты · сырые эндпоинты/инъекция в систему.
**Принцип:** конфигурируешь ПОВЕДЕНИЕ — не получаешь ИСПОЛНЕНИЕ КОДА или ДАННЫЕ сверх своей песочницы. Клиенту не верим — всё проверяем на сервере по whitelist.

## ⚙️ ОБЩАЯ СХЕМА КОНФИГА (контракт для всех рабочих)
```
userConfig[mode] = {
  model?: string,          // какая модель на этот режим
  maxTokens?: number,      // длина/цена ответа (кап серверным потолком)
  temperature?: number,    // 0..1.5
  systemPrompt?: string,   // персона/инструкция на режим (скрабится)
  tools?: { [name]: bool } // какие инструменты доступны (только из безопасного списка)
}
customFunctions[] = {
  id, name, icon, pill: true,
  base: "council"|"brain"|"relay"|"research"|"compare"|"image"|"web"|...,  // примитив
  params: { models?: number, iterations?: number, depth?: ..., compareModels?: [], ... },
  config: userConfig-подмножество
}
```
Хранится per-user (SQLite). Применяется в `chat()`. UI правит — бэкенд валидирует+применяет.

## 🗺️ КАРТА КОНФИГА (каждая функция → ручки юзера = «100 кнопок»)
| Функция | Кнопка/окно (как у Мозга) | Ручки юзера |
|---|---|---|
| **Мозг** | ✅ есть | эксперт: выбранная/умная · показ шагов · (+) модель-ведущий, промпт |
| **Связка (Улучшить)** | ✅ есть (craft-текст) | модель-переписчик · инструкция · модель-ответчик |
| **Агент** | ✅ есть (ведущий/эксперт) | + сохранить как пилюлю |
| **Совет** | ⬜ построить | число моделей · какие модели · синтез-модель |
| **Сравнение** | ✅ есть (мультивыбор) | какие модели · сколько |
| **Ресёрч** | ⬜ построить | глубина · число источников · модель |
| **Студия** | частично | модель img/video/music · качество · free/paid |
| **Каждый режим (Fast/Expert/…)** | ⬜ построить | модель · длина · температура · промпт · инструменты |
| **Своя функция** | ⬜ ГЛАВНОЕ | собрать из примитивов + параметры → своя пилюля + своё окно |

## 🌊 СБОРКА — ВОЛНА 2 (4 параллельных рабочих, независимые поверхности)
- **W1 BACKEND-CONFIG** (`server.py`): per-user safe-config store (SQLite) + apply в chat() + `/api/userconfig` (whitelist-валидация, security-гейт) + `/api/build_function` (ИИ строит конфиг по NL).
- **W2 FUNCTION-BUILDER** (`lib/commands/*` + новый компонент, БЕЗ chat.tsx): конструктор «собери из примитивов + параметры» → user-store; поле «опиши словами» → /api/build_function.
- **W3 DEV-MODE-PANEL** (`settings-panel.tsx` + новый компонент): раздел «Режим разработчика» — дефолты на режим (модель/длина/темп/промпт/инструменты), пишет userConfig.
- **W4 PER-MODE-WINDOWS** (`chat.tsx`): каждой пилюле — шестерёнка → окно безопасных ручек (как у Мозга); рендер кастом-пилюль из store.

Все 4 — РАЗНЫЕ файлы → параллельно без коллизий. Контракт = схема выше. Координатор сливает + проверяет security-гейт.

## ПОЧЕМУ НЕ 40 АГЕНТОВ СРАЗУ (честно)
Предел — не желание, а **независимые поверхности**: `chat.tsx` = 3491 строк, один god-файл, его может пилить ТОЛЬКО один агент за раз (иначе merge-ад). Worktree-изоляция позволяет 4+ на РАЗНЫХ файлах. Будущий рефактор chat.tsx на модули → ещё больше параллели.
