# Тайга ИИ — план (React-ребилд, best practices 2026)

## Что это
Приватный мульти-модельный ИИ-чат (RU-рынок). Переезжаем с ванильного HTML на полноценный React-стек.
Дизайн пока вторичен («просто красивый фон»), фокус — функции и архитектура.

## Зафиксированные решения (с Дамиром)
1. **5 новых режимов заменяют старые 5.** Старые (мозг/связка/совет/ресёрч) → внутренние функции, которыми ИИ пользуется сам.
2. **Коробка передач:** одна беседа, Тайга авто-переключает режим (модель+инструменты+поведение+фон) по намерению. Пилюли = индикатор + ручной фиксатор. «Без цензуры» = sticky-тумблер приватности.
3. **У каждого режима свои функции** (Код: запуск/дифф/тест; Картинки: генерация; Ультра: совет+ресёрч; и т.д.).
4. **Сильный агент со step-by-step** (план по шагам вживую, статусы, какие инструменты дёргает).
5. **Кастом-функции юзера:** (D1) команды-шорткаты (промпт-пресеты), (D2) пайплайны «модель→модель→модель».
6. **Безопасность = пермишены как в Claude Code:** чувствительное действие → Разрешить раз / Всегда / Запретить. Приватные режимы — без логов.
7. **Весь фронт — React.** Ваниле конец.

## Стек (best practice)
- **Язык:** TypeScript (фронт + бэк)
- **Фреймворк:** Next.js 15 (App Router) — фронт + API-роуты + стриминг в одном
- **UI:** Tailwind CSS + shadcn/ui + Framer Motion (компоненты 21st-dev/shadcn встают нативно)
- **AI-слой:** Vercel AI SDK (`ai`) — стандарт для AI-апп: стриминг, tool-calling, мульти-модель, `useChat`
- **Провайдеры моделей:**
  - Чат: Anthropic (Claude), OpenAI, Google (Gemini)
  - Код: qwen3-coder (Together/Groq) / Claude
  - Без цензуры: **Venice** (OpenAI-совместимый, без логов)
  - Картинки: **Nano Banana Pro** (Gemini 3 Pro Image) + FLUX (fal.ai)
  - Видео: Kling / Veo 3.1 + **Remotion** (видео из данных через Opus)
  - Ультра: совет нескольких моделей + веб-ресёрч
- **БД:** Supabase (Postgres + Auth + Storage + pgvector) — аккаунты, история, память (embeddings), кастом-функции; RLS под безопасность
- **ORM:** Drizzle (типобезопасно) или Supabase client
- **Песочница кода:** E2B (режим Код)
- **Деплой:** Vercel (фронт+API) + Supabase (БД)
- **Python `server.py`:** оставляем как модель-сервис на переход; логику роутинга портируем в TS постепенно (не теряем рабочее)

## Открытый вопрос
- Глубина порта бэкенда: полностью в Node/AI SDK vs Python-сервис на переход. План: фронт+API на Next сразу, Python дёргаем как сервис, портируем по режимам.

## Стройка по шагам
1. **Каркас** — Next.js + TS + Tailwind + shadcn + Framer Motion; чат-экран; стриминг через AI SDK; прокси/мост к существующему `/api/chat`.
2. **Коробка передач** — авто-детект намерения + ручной лок + смена фона per-режим.
3. **Step-by-step агент** (agent-plan UI).
4. **Кастом-функции** — шорткаты + пайплайны «модель→модель».
5. **Пермишены** (Claude-Code-style) + приватные режимы без логов.
6. **БД/аккаунты** на Supabase, память на pgvector.
7. Дизайн-полировка по режимам (фоны, particle-text, spotlight, liquid-text) — в конце.

## Режим Картинки/Видео = генеративная студия (по мотивам open-generative-ai)
Ориентир: anil-matcha/open-generative-ai (uncensored, self-hosted, 200+ готовых моделей через шлюз muapi.ai).
Тулзы режима (на готовых моделях, без цензуры):
- **text→image** (Nano Banana Pro / FLUX / Ideogram / GPT-4o-image)
- **image→image** (правка, до N реф-картинок)
- **text→video** (Kling / Veo / Wan / Seedance)
- **image→video**
- **lip-sync** (анимация портрета) — позже
- **upscale / варианты**
- **node-пайплайны** «модель→модель» — пересекается с кастом-функциями юзера (D2)
Подключение: единый шлюз (muapi.ai «один ключ — 200 моделей» ИЛИ fal.ai) — чтобы не вести 10 вендоров.
Безопасность: платная генерация → пермишен (Разрешить раз/Всегда), приватный режим без логов.
Статус: отдельный большой милстоун (после авто-передачи). Нужен ключ шлюза + решение по бюджету.

## Каркас функций (общие + кастомные по режимам)
«Функция» = возможность внутри режима (под-режим/тулза), вызывается кнопкой/командой.

ОБЩИЕ функции (во всех режимах, кнопки в композере):
- **План** — раскладывает задачу на шаги (step-by-step, твой agent-plan UI)
- **Глубоко / Think** — глубокое размышление (reasoning, макс. усилие)
- **Ресёрч** — веб-поиск + свод (есть в backend)
- **Память** — контекст/факты

КАСТОМНЫЕ функции по режимам:
- **Чат** — минимум (оригинал): ответ + общие функции. НЕ перегружаем (как просил).
- **Код** — code-skills из claude-skills (архитектура/тесты/devops/секьюрити) + **Терминал** + **Запуск локально** + Дифф + Гит. ⚠ shell/run — только владелец (анти-RCE, уже в backend).
- **Без цензуры** — raw: функции Чата+Кода, без цензуры, без логов.
- **Картинки/Видео** — генерация (t2i/i2i/t2v/i2v/lip-sync/upscale) + пайплайны (open-generative-ai).
- **Ультра** — Ультра-воркер (мульти-агент) + пошаговый планировщик + совет моделей (LLM council) + deep research. Строим сами (council/research уже есть), claw-code НЕ используем (музейный экспонат).

Источники методологии: claude-skills (формат SKILL.md + тулзы), open-generative-ai (медиа-студия). Пермишены Claude-Code-стиля на всё чувствительное.

## Агентные способности (файлы / терминал / браузер / PC — «как Claude Code»)
Честная архитектура (граница возможностей):
- **Веб-приложение (браузер) НЕ МОЖЕТ** само лазить по реальным файлам/браузеру юзера — это песочница браузера. Может: upload (через пикер), download, + работать в ИЗОЛИРОВАННОЙ облачной песочнице.
- **Фаза 1 — песочница E2B (веб, делаем первой):** реальный терминал + запуск кода + чтение/запись/правка файлов В ПЕСОЧНИЦЕ + скачивание результата. Это и есть «делает дела» как Claude Code, но безопасно (изолировано от реального ПК). Нужен ключ E2B (есть free-tier).
- **Фаза 2 — десктоп-приложение (Tauri/Electron) + расширение браузера:** доступ к РЕАЛЬНЫМ файлам/терминалу/браузеру юзера. Полный «Claude Code на своём ПК». Другая дистрибуция.
- **Генерация:** картинки — РАБОТАЮТ через venice-sd35 (✅ проверено, ключ Venice). Видео — нужен видео-модель/шлюз.
- На всё чувствительное — пермишены Claude-Code-стиля (Разрешить раз/Всегда/Запретить).
