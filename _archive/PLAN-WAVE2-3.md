# План «лучший подход» к Волнам 2-3 — без клэшей, низкий риск (ural/BACKEND)

## ГЛАВНЫЙ ТЕЗИС
«Переезд на SQLite» как полная миграция — **НЕПРАВИЛЬНАЯ цель**: рискованно (kedr/amur сидят на
JSON), долго, ломает других. Реальные проблемы, которые SQLite «решил бы», — **точечные**, и
чинятся surgical-фиксами БЕЗ миграции. Один уже сделан (гонка баланса). Принцип всей Волны-2/3:
**аддитивно (новые функции/эндпоинты/демоны) + пофайловый лок + отдать UI в CORE.**

## КООРДИНАЦИЯ (как не клэшнуться)
- `server.py` — общий (BACKEND ядро + STUDIO ген-функции `charge_media/api_image/...`). Перед КАЖДОЙ
  правкой: `claim "server.py" "ural"` → правлю быстро → `release`. Если BUSY:amur — беру другую задачу.
- Новые МОДУЛИ (auth.py, hooks.py, scheduler.py) — отдельные файлы, ноль коллизий, лок не нужен.
- UI к новым эндпоинтам — лана CORE (kedr), контракты в BACKEND-API.md. Я бэкенд, не лезу в .tsx.
