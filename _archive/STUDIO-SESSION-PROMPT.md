## РЕАЛЬНОСТЬ ПО ДЕНЬГАМ И ПРОВАЙДЕРАМ (важно — это поменялось)

Балансы провайдеров (видны фронту через /api/init → balances, и на бэке через get_balances()):
- Chutes  ≈ $10   (есть деньги)
- Venice  ≈ $4.88 (есть деньги)
- RedPill ≈ $4.84 (есть деньги; баланс по API не отдаётся — считаем доступным)
- NanoGPT ≈ $0.04 (ПУСТО) — НО активна подписка: 100 картинок/день бесплатно + 60М входных
  токенов/нед. Подписка покрывает ВХОД, выход — с баланса.

ПРАВИЛО ПРОВАЙДЕРА (жёсткое): только крипто-оплата + перепродажа разрешена.
Подходят: NanoGPT, Venice, Chutes, RedPill. Для видео/музыки/3D одним пулом — AIMLAPI
(Damir заливает стартовый $20 криптой сам; не предлагай Stripe/карты как обязательное).
muapi и подобные без крипто/ресейла — НЕ использовать.

## ЧТО СДЕЛАТЬ В СТУДИИ (применить те же принципы, что в тексте)

1. БАЛАНСО-ОСОЗНАННЫЙ выбор медиа-модели. Не отправляй генерацию на провайдера с пустым
   кошельком. Если у выбранной медиа-модели провайдер пустой — бери ту же/аналогичную модель
   у провайдера с деньгами (как кросс-провайдерный роутер в тексте). Балансы бери из
   /api/init → balances (порог «есть деньги» > $0.2; no_api_balance/usd=null → считать доступным).

2. КАРТИНКИ — НА ДВУХ ПРОВАЙДЕРАХ (не только NanoGPT!):
   - NanoGPT: 191 модель, /v1/images/generations. hidream / qwen-image входят в подписку
     (NANO_SUB_IMAGE_MODELS, 100/день, $0 СЕБЕСТОИМОСТИ ВЛАДЕЛЬЦА). nano_sub_status() это знает.
   - Venice: 30 моделей, /api/v1/image/generate + /image/upscale + /image/edit (фото-тулзы).
     Venice ОПЛАЧЕН (~$4.88) → его картинки работают прямо сейчас.
   ВАЖНО: «бесплатно по подписке» = $0 СЕБЕСТОИМОСТИ ВЛАДЕЛЬЦУ, НЕ бесплатно юзеру. Юзер
   ПЛАТИТ как обычно (charge_media с наценкой) — подписка просто даёт высокую маржу, не дисконт
   юзеру. Никогда не делай генерацию бесплатной для юзера из-за подписки.
   Дефолт = hidream (NanoGPT, $0 себестоимости). Если NanoGPT не отвечает — падай на Venice
   (оплачен). Балансо-осознанно: пустой провайдер не выбираем.

3. ВИДЕО — ПОКА ТОЛЬКО NanoGPT (/api/generate-video), и баланс ≈ $0 → платное видео упадёт
   с 402. Venice/Chutes/RedPill видео НЕ делают. Поэтому для видео (и музыки/3D) подключи
   AIMLAPI пулом (крипто+ресейл). До заливки пула — показывай юзеру понятную причину
   («видео-провайдер не пополнен»), НЕ безликое «недоступно».

3b. АУДИО/TTS — пока только NanoGPT (/api/tts). Venice тоже умеет TTS (/api/v1/audio/speech) —
   можно добавить вторым провайдером озвучки (оплачен), чтобы не зависеть от NanoGPT-баланса.

3c. TASTE LEARNING (идея из gstack — killer-фича студии): запоминай лайки/дизлайки генераций
   юзера (вес затухает ~5%/нед) и смещай будущие генерации под его вкус (модель/стиль/промпт-
   добавки). Локальное хранилище, как chat-memory. Это сильно поднимает ценность студии.

4. ПОНЯТНЫЕ ОШИБКИ. Если провайдер вернул 402/insufficient_balance — показывай
   «кончился баланс провайдера», 502 — «провайдер недоступен», 429 — «лимит». Не прячь за
   «генерация не удалась».

5. НАЦЕНКА/РЕСЕЙЛ уже есть на бэке (charge_media с markup) — НЕ дублируй биллинг, пользуйся им.
