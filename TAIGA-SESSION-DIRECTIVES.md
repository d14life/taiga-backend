Свод от оркестрации (2 разбора + синтез) + внешний аудит. Для каждой сессии: вердикт + готовый промпт. В конце — правила, чтобы сессии не столкнулись.

# СЕССИЯ 1 — СТУДИЯ ГЕНЕРАЦИИ

## Вердикт — 72/100

Студия **реально работает end-to-end по всем 5 режимам** (картинки Venice, видео NanoGPT submit→poll, аватар/тулзы, TTS). Движок честный: carry-буфер SSE, AbortController-стоп, живой прогресс видео, умный дефолт image-моделей.

**🔴 Красная денежная дыра — картинки НЕ списываются с баланса вообще.** Image-ветка делает `return` (`server.py:3108`) ДО `meter()` (живёт в чат-цикле `:3216-3229`). Цена картинки известна (`gen_usd`), но не списывается. Гейт только «баланс > 0» → заплатил раз → генеришь картинки бесконечно бесплатно, ещё и параллельно по `count`. Прямая утечка маржи с первого платящего юзера. Видео-списание тоже дырявое (списывает 0 если цена не пришла, баланс уходит в минус, тихий `except: pass`).

**Декоративные контролы (врут юзеру):** img2img принимается и игнорируется (`image-studio.tsx:202`); негатив-промпт не уходит в `negative_prompt` Venice, а склеивается в текст (`:207`); aspect картинки зашит 1024×1024 (`server.py:1155`), чип меняет только CSS-рамку. История картинок/TTS эфемерна (фильтр режет base64).

**Главный риск:** генерация картинок не тарифицируется = потеря денег.

# СЕССИЯ 2 — ОСНОВНОЕ ПРИЛОЖЕНИЕ (ЯДРО / АГЕНТНАЯ ОС)

## Вердикт — 68/100 (интеллект ~85, инфра ~45)

Интеллект **сильный и настоящий**. Model-router (`select.ts`) — лучшее место в проекте: GATE → quantile-пол → SCORE → primary + 3 фолбэка разных брендов. relay/brain/council/research/pipeline реальны, BYOK везде проброшен. Память модель-нейтральная с анти-отравлением.

**Весь долг в STATE/INFRA — это потолок. Подтверждены все 5 пунктов аудита:**
1. Сообщения = **плоский список** `ChatMessage[]` без parent/children (`use-taiga-chat.ts:12-19`); edit/regenerate режут через `slice` и теряют старую версию навсегда (`chat.tsx:952-970`). Ветки/чекпоинты/replay архитектурно невозможны.
2. **БД нет** — всё в плоских JSON; balance.json read-modify-write без локов → параллельные запросы перезатирают баланс; чаты только в localStorage (чистка кэша = потеря всего).
3. SSE-события только `meta/delta/tool/research_step/council` — **нет** `plan/permission/verify/subagent/run_done`. Агентный цикл невидим.
4. **Permission ladder ненастоящая** — `permissions.ts` это frontend-only ACL, гейтит ТОЛЬКО slash-команды; реальный вызов тулзы агентом на бэке (`server.py:3193`) НЕ проверяется. Права = косметика.
5. **`test_topup=True`** (`server.py:2491`), реальный процессинг не написан.

**🔴 Второй денежный launch-blocker: charge-after-stream + пред-чек `<= 0`.** Юзер с балансом $0.0000001 проходит гейт и запускает сколь угодно дорогой council/research — уходит в минус, спишется после. Нет резервирования. Плюс «sandbox» = `subprocess.run(shell=True)` на хосте, owner-gate на хрупком `uid=="default"`.

**Главный bottleneck прямо: плоский список сообщений.** Пока не дерево — нет ветвления/чекпоинтов/сабагентов/replay.

# КАК НЕ СТОЛКНУТЬ ДВЕ СЕССИИ

1. **Жёсткое разделение по файлам.** Студия владеет: `image-studio.tsx`, `gen-image.ts`, `video-gen.ts`, `audio-gen.ts`, `app/api/(chat|video|audio|video-models)/*`. Ядро владеет: `chat.tsx`, `use-taiga-chat.ts`, `chat-memory.ts`, `permissions.ts`, новый слой БД/event-log. Не пересекаются — внутри работайте свободно.
2. **`server.py` делится по функциям, не по строкам.** Студия редактирует ТОЛЬКО функции генерации и их метеринг. Ядро — agent-loop, billing/pre-reserve, storage, auth. Функцию списания (`:1945-1963`) меняет ТОЛЬКО ядро; студия её ВЫЗЫВАЕТ, не меняет сигнатуру.
3. **`select.ts` и `package.json` — фриз для студии.** Правит только ядро. Студии нужна зависимость/поле — заявка ядру.
4. **При конфликте — приоритет у ветки ядра** (фундамент), студия ребейзится. Метеринг: студия чинит списание картинок/видео СЕЙЧАС через вызов существующей функции (не плодя свою логику баланса); ядро ПОТОМ заворачивает весь биллинг в транзакции.
