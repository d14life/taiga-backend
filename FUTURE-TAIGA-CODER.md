# ФЛАГМАН v2: «Тайга-Кодер» — Jules/Devin внутри Тайги (Damir-vision 2026-06-12)

Свой автономный ИИ-кодер: юзер даёт задачу + репо → Тайга планирует → правит файлы → гоняет тесты в изолированной среде → открывает PR. Приватно, мульти-модель, крипто-оплата, ресейл-OK.

## ЧТО УЖЕ ЕСТЬ (~70%)
- 🧠 **Мозг**: Gemini 3.1 Pro + 2.5-flash + любая модель (мульти-провайдер). ✅
- 🔗 **GitHub**: MCP-коннектор (`api.githubcopilot.com/mcp` — репо/issues/PR) в маркете. ✅
- ⌨️ **Исполнение кода**: `run_code` (python/js/bash), `tool_shell`, `run_code_lang` (rlimits+temp-cwd). Код сам помечает: «полная мульти-тенант изоляция = контейнеры/E2B на деплое». ✅(локально) / нужна изоляция для юзеров.
- 📁 **Файлы**: read_file/write_file/edit_file/revert (DEV_TOOLS, owner-gated). ✅
- 🧷 **Память**: Mem0/Letta (рабочая память агента между шагами). ✅
- 🎛 **Оркестратор + конструктор агентов**. ✅

## ГЛАВНЫЙ ВОПРОС — ГДЕ ИСПОЛНЯТЬ КОД (4 модели, юзер выбирает):
1. **Управляемая песочница** (E2B / Daytona / Modal / Fly Machines) — мы поднимаем эфемерную микро-VM на задачу, рвём после. Pay-per-use → берём с юзера криптой. Так делают Jules/Devin. Лучший UX, мы контролируем. Минус: наша compute-стоимость.
2. **Свой сервер юзера (BYO)** — юзер даёт SSH к своему VPS → Тайга гоняет там. Юзер платит за железо, макс. контроль+приватность, нам compute = 0. Для power-юзеров.
3. **Локальный демон («домашний компилятор»)** — мини-агент Тайги на машине юзера (как Claude Code), на ЕГО железе. Бесплатный compute, макс. приватность. Идеально под приват-рынок РФ.
4. **Эфемерно без хранения** — песочница встала → отработала → вывод в чат → снесена, НИЧЕГО не сохранено (приват-режим). Для быстрых прогонов.

## ФАЙЛЫ / ПАМЯТЬ
- Во время задачи — в ФС песочницы/VM. Сохранить → push в GitHub (MCP/git) ИЛИ облако ИЛИ локально юзера. Не хранить → эфемерно (приват).
- Память агента (план, тронутые файлы, результаты тестов, решения) — через наш Mem0/Letta слой.

## 🔐 БЕЗОПАСНОСТЬ (обязательно)
Исполнение НИКОГДА не на нашем основном сервере. Всегда изолированная per-user песочница (анти-RCE). Токены GitHub — аккуратно, не логировать. Это самый security-тяжёлый кусок.
